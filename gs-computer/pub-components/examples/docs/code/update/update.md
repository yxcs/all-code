## 组件库更新规范

组件库更新之前，要确定有没有更新的必要！

### 更新代码库

1. 切换新分支
2. 代码更改保存
3. 推送到远程
4. 合并到master分支
5. 打本次更新的版本标签

### 插件库更新

1. 更改 `package.json` 中的版本号
2. 运行 `npm run build` 构建最新插件
3. 推送到内网NPM服务器：`npm publish --registry=http://192.168.99.117:7001/`

### 组件文档更新

1. 编写相应的demo文档
2. 运行 `npm run build_example` 构建最新的文档应用
3. 将 `dist` 文件夹发送服务器
4. 刷新文档地址，查看最新文档内容

### 版本号管理

版本号规则：主版本号.次版本号.修补版本号

根据本次更改的范围确定版本号
